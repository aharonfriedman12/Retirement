# אפליקציית ניהול משימות — פרישה מצה״ל

אפליקציית React אישית שמתחברת לחשבון הגוגל שלך, מנהלת את המשימות בגיליון Google Sheets,
ויוצרת תזכורות ביומן גוגל. הכול רץ מהדפדפן — אין שרת, והנתונים נשארים אצלך.

> סדר הפעולות: **שלב 1 (גיליון) → שלב 2 (Google Cloud) → שלב 3 (הדבקת 2 ערכים) → שלב 4 (הרצה) → שלב 5 (פרסום)**

---

## שלב 1 — הכנת גיליון גוגל שיטס

1. צור גיליון חדש ב-https://sheets.google.com
2. צור **שתי לשוניות** עם השמות המדויקים `Tasks` ו-`Config` (אותיות גדולות, באנגלית).
3. בלשונית **`Tasks`**, הדבק בשורה הראשונה (A1) את הכותרות הבאות, כל אחת בתא נפרד:

```
id	title	cat	status	prio	due	reminderDays	who	phone	docs	link	note	doneDate	eventId
```
(העתק-הדבק את השורה לתוך תא A1 — גוגל יפזר אוטומטית לעמודות. בעברית: מזהה, שם, קטגוריה,
סטטוס, עדיפות, תאריך יעד, ימי תזכורת, גורם מטפל, טלפון, מסמכים, קישור, הערות, תאריך השלמה, מזהה אירוע.)

> חשוב: עמודות `due` ו-`doneDate` בפורמט תאריך `YYYY-MM-DD` (למשל 2026-10-15). אפשר להוסיף משימות גם מתוך האפליקציה.

4. בלשונית **`Config`**, הדבק בעמודה A/B (החל מ-A1):

```
key	value
dischargeDate	2026-10-15
journeyStart	2026-03-01
```
שנה את התאריכים שלך: `dischargeDate` = תאריך הפרישה, `journeyStart` = מתי התחלת את התהליך (משמש לחישוב אחוז ההתקדמות).

5. שמור את **מזהה הגיליון** מתוך כתובת ה-URL:
`https://docs.google.com/spreadsheets/d/`**`<<< החלק הזה >>>`**`/edit`

## שלב 2 — הקמת פרויקט ב-Google Cloud (כ-10 דקות)

הממשק כיום נקרא **Google Auth Platform**. היכנס ל-https://console.cloud.google.com

1. **פרויקט חדש**: בורר הפרויקטים למעלה → "New Project" → תן שם (למשל `retirement-app`) → Create.
   ודא שהפרויקט החדש נבחר בבורר למעלה.

2. **הפעלת ה-APIs**: תפריט → `APIs & Services` → `Library`. חפש והפעל (Enable) **שני** שירותים:
   - `Google Sheets API`
   - `Google Calendar API`

3. **הגדרת מסך ההסכמה (Auth Platform)**: בתפריט → `Google Auth Platform` (או היכנס ל-`APIs & Services` → `OAuth consent screen`). אם זה פרויקט חדש, לחץ **Get started** ועבור את האשף:
   - App name: שם לבחירתך · User support email: המייל שלך
   - **Audience: בחר `External`** (חשוב! בלי זה אפליקציה אישית לא תעבוד)
   - Contact email: המייל שלך → סיים את האשף.

4. **הוספת עצמך כמשתמש בדיקה**: ב-`Google Auth Platform` → `Audience` → תחת **Test users** לחץ Add users והוסף את כתובת הג'ימייל שלך. (כל עוד האפליקציה ב-"Testing", רק משתמשי בדיקה יכולים להתחבר — וזה בדיוק מה שאנחנו רוצים לשימוש אישי.)

5. **יצירת Client ID**: `Google Auth Platform` → `Clients` → **Create client**:
   - Application type: **Web application**
   - Name: לבחירתך
   - תחת **Authorized JavaScript origins** הוסף (Add URI):
     - `http://localhost:5173`  ← לפיתוח מקומי
     - את כתובת ה-Vercel שלך תוסיף כאן בהמשך (שלב 5)
   - **אין צורך** ב-Authorized redirect URIs עבור שיטת ההתחברות הזו.
   - Create → **העתק את ה-Client ID** (מסתיים ב-`apps.googleusercontent.com`).

## שלב 3 — הדבקת שני הערכים

פתח את `src/config.js` והדבק:
- `CLIENT_ID` = ה-Client ID משלב 2.5
- `SHEET_ID` = מזהה הגיליון משלב 1.5

זהו — אין עוד מה לערוך שם (אלא אם תרצה לשנות קטגוריות/צבעים).

## שלב 4 — הרצה מקומית

צריך Node.js מותקן (https://nodejs.org). בטרמינל, בתוך תיקיית הפרויקט:

```bash
npm install
npm run dev
```
פתח את הכתובת שמודפסת (בד"כ http://localhost:5173). לחץ "התחברות עם גוגל",
אשר את ההרשאות, והדשבורד ייטען מהגיליון שלך.

> בפעם הראשונה גוגל יראה אזהרת "אפליקציה לא מאומתת" — זה תקין לאפליקציה אישית.
> לחץ Advanced → Continue. ההרשאות מוגבלות לגיליון וליומן שלך בלבד.

## שלב 5 — פרסום ב-Vercel (כדי לגשת מהנייד)

1. העלה את הקוד ל-GitHub (repo חדש).
2. ב-https://vercel.com → New Project → ייבא את ה-repo. Vercel מזהה Vite אוטומטית — לחץ Deploy.
3. קבל כתובת כמו `https://retirement-app-xxxx.vercel.app`.
4. **חזור ל-Google Cloud** → Clients → ערוך את ה-Client → הוסף את כתובת ה-Vercel תחת
   **Authorized JavaScript origins** → Save. (בלי זה ההתחברות תיכשל בכתובת החדשה.)
5. בנייד: פתח את הכתובת ב-Chrome/Safari → "הוסף למסך הבית". זה ייראה ויתנהג כמו אפליקציה.

## מה עובד עכשיו (גרסה 1)
- התחברות מאובטחת לחשבון שלך (OAuth)
- דשבורד חי: ספירה לאחור, דחוף/באיחור/ממתין, התקדמות לפי קטגוריה
- קריאה וכתיבה אמיתית לגיליון (שינוי סטטוס, הוספת משימה)
- יצירת/עדכון תזכורת ביומן גוגל (סנכרון חד-כיווני אפליקציה→יומן)
- מסכי משימות / מסמכים / אנשי קשר

## שלבי המשך מתוכננים (גרסה 2)
- **התראות פוש** מהאפליקציה (Service Worker + Web Push)
- **סנכרון דו-כיווני** ליומן (זיהוי שינויים שבוצעו ביומן והחזרתם למשימה)
- עריכת משימה קיימת מתוך האפליקציה (כרגע עריכה נוספת נעשית בגיליון)

## פתרון תקלות
- **"redirect_uri_mismatch" / חלון נסגר מיד**: כתובת הגישה לא נמצאת ב-Authorized JavaScript origins. הוסף אותה בדיוק (כולל http/https והפורט).
- **"access_denied"**: לא הוספת את עצמך כ-Test user (שלב 2.4).
- **403 בקריאה לגיליון**: לא הפעלת את Google Sheets API, או שה-SHEET_ID שגוי.
- **השמות של הלשוניות**: חייבים להיות בדיוק `Tasks` ו-`Config`.
